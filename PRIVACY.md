# Privacy And Safety

Please keep all intake issues safe for public GitHub.

Do not include:

- Passwords.
- API keys.
- Recovery codes.
- Private keys or seed phrases.
- Customer data.
- Internal URLs.
- Production credentials.
- Bank, tax, identity, or payout details.

If a repository or workflow needs private access, mention that private access is required without posting sensitive details. Scope and access can be handled after the request is accepted.
